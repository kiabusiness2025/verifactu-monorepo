# Preparación para voz (futuro)

## Objetivo
Permitir que el usuario dicte un gasto y que Isaak lo valide, clasifique y registre.

## Principio clave
La voz es solo una entrada. El flujo final siempre es el mismo:
1) Captura de datos
2) Validación por Isaak
3) Registro en base de datos

## Datos mínimos
- Fecha
- Importe
- Descripción

## Datos deseables
- Proveedor
- Referencia
- Impuestos

## Respuesta de Isaak
- Categoría sugerida
- Deducible: sí/no
- Registro final confirmado
